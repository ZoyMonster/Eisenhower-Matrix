# 图标转换说明

## 问题
如果图标没有显示，可能是因为 `build/icon.icns` 文件格式不正确（可能是 JPEG 文件被重命名为 .icns）。

## 解决方案

### 方法一：使用在线工具（最简单）

1. 准备一个 **1024×1024 像素的 PNG 图片**（或使用你现有的图片）
2. 访问在线转换工具：
   - https://cloudconvert.com/png-to-icns
   - https://iconverticons.com/online/
3. 上传 PNG 图片，下载生成的 `.icns` 文件
4. 将下载的文件重命名为 `icon.icns`，放到 `build/` 目录下
5. 重新打包：`npm run build:dmg`

### 方法二：使用 macOS 预览应用

1. 用"预览"打开你的图片（PNG 或 JPG）
2. 选择"文件" → "导出"
3. 格式选择 **"ICNS"**
4. 保存为 `build/icon.icns`
5. 重新打包：`npm run build:dmg`

### 方法三：使用命令行（如果图片是 PNG）

```bash
# 确保有 PNG 源文件（例如 icon.png）
mkdir -p icon.iconset

# 生成各个尺寸
sips -z 16 16     icon.png --out icon.iconset/icon_16x16.png
sips -z 32 32     icon.png --out icon.iconset/icon_16x16@2x.png
sips -z 32 32     icon.png --out icon.iconset/icon_32x32.png
sips -z 64 64     icon.png --out icon.iconset/icon_32x32@2x.png
sips -z 128 128   icon.png --out icon.iconset/icon_128x128.png
sips -z 256 256   icon.png --out icon.iconset/icon_128x128@2x.png
sips -z 256 256   icon.png --out icon.iconset/icon_256x256.png
sips -z 512 512   icon.png --out icon.iconset/icon_256x256@2x.png
sips -z 512 512   icon.png --out icon.iconset/icon_512x512.png
sips -z 1024 1024 icon.png --out icon.iconset/icon_512x512@2x.png

# 转换为 ICNS
iconutil -c icns icon.iconset -o build/icon.icns

# 清理
rm -rf icon.iconset
```

## 验证图标文件

打包前可以验证图标文件格式：
```bash
file build/icon.icns
```

应该显示类似：`build/icon.icns: Apple Icon Image format`

如果是 JPEG 或其他格式，说明文件不正确，需要重新转换。
